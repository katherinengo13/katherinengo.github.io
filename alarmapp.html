iPhone Alarm App Redesign

"A useful application that is used everyday, but it could be better."

For a design project, my group and I decided to analyze the different errors and slips people make when using the iPhone Alarm App so that we could redesign different aspects of the app's usability.

Note: I continued the project by creating high fidelity prototypes using Figma.

Role: UX Researcher and Designer

Data Collection

We decided to use an interview methodology as well as ask the interviewees to perform a list of actions. Before following the list of instructions, we asked each interviewee a couple of questions.

Interview Questions(Pre-activity)

1. Do you own an Android or an iPhone?
2. Do you use the alarm on the phone?
3. Have you used the iPhone clock app as an alarm before?
a. If so, how often?
4. Why/why do you not use an alarm?
a. If not, what do you usually use as an alarm?
b. How often?
5. What are you general thoughts on the iPhone clock app?

We asked interviewees if they used an Android or iPhone because those who own iPhones would naturally know more about the functionality and interface of the alarm app versus someone who owns an Android. When we aked if they have used an iPhone alarm, we wanted to gauge whether they had background knowledge or experience with the app which would therefore affect how they use the app.

List of Instructions

We wrote out these different steps of instructions because we wanted to cover all the different functions of the app and observe how the interviewees interact with these different functions.

Interview Questions(Post-activity)

After the interviewees performed the list of actions, we followed up with more questions.

1. After completing these steps, what are your general thoughts about the app?
2. If they made any specific errors/actions, we would ask them why they think they made that error/action.

Data

Our group interviewed a total of 14 people, which consisted of 8 iPhone users and 6 Android users. We analyzed different errors that seemed to occur during the interviews and kept count of how many interviewees made each error.

Error Analysis

Interviewee tapping on the bar to try to edit the alarm

Interviewee making a slip and scrolling AM or PM when they really wanted to scroll through minutes

Interviewee tapping on the wrong day

Redesign

For our redesign, we focused on 3 different aspects on the app to redesign.

Wireframes for the redesigns.

Checking off multiple alarms.

For our first redesign, we decided to design a function of deleting multiple alarms at once. This function is based off a checklist so that the user can efficiently delete alarms. The user simply presses "Edit" and then the check boxes appear so that the user can select the alarms and then delete them all at once when the user clicks on the trash can icon.

Tap to edit the alarm.

The most common error that interviewees made trying to tap the alarm to edit it. So for our redesign, we propose that rather than pressing 'edit' then choosing the alarm the user wishes to edit, the user should be able to press the alarm directly and enter the edit settings of the alarm. This would simplify the layout and reduce the amount of tapping users would have to do in order to edit an alarm.

Typing in the time.

Since one of the most common errors people made was accidentally scrolling past certain times they meant to set for their alarms, we decided to redesign the way users set the times for their alarms. Instead of incorporating a scrolling function, our allows the user to type in the time. This enables the user to ensure precision.
